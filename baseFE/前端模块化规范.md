## 模块
- 将一个复杂的程序依据一定的规则(规范)封装成几个块(文件), 并进行组合在一起
- 块的内部数据与实现是私有的,只是向外部暴露一些接口(方法)与外部其它模块通信

## 模块规范

### CommonJS

- 基本语法
  - 暴露模块/引入模块：module.export/requere(xxx) 

- 特征
  - 加载方式：require 为同步操作，不适用与浏览器环境
    - 浏览器：模块存放在服务器，读取速度取决于网速
    - 服务器端：模块存放在本地硬盘，读取速度即硬盘读区速度
  - 执行时机：运行时
  - 模块
    - 模块输出：值的拷贝
    - 模块引用：第一次加载时运行一次，会缓存值

### ES 模块化

- 基本语法
  - 暴露模块/引入模块：export/import


- 特征
  - 执行时机：编译时确定模块的依赖关系
  - 模块
    - 模块输出：输出值的引用
    - 模块引入：动态引入，不会缓存值（ 类似于 Unix 系统的“符号连接”）


- 注意
  - export & export default(查看 demo1)
    - export（命名式导出）：导出的是引用
    - export default（默认导出）： 导出的是值

### AMD

- 特征
  - 加载方式：异步加载模块
  - 执行时机：依赖前置，提前执行

### CMD

- 特征
  - 加载方式：异步加载模块
  - 执行时机：依赖就近，延迟执行

### UMD

- 特征
  - 加载方式：根据应用环境进行判断，UMD 是 AMD 和 CommonJS 的糅合



> demo1

```
// a.js
let x = 10
let y = 20
setTimeout(()=>{
    x = 100
    y = 200
},100)
export { x }
export default y

// b.js
import { x } from './a.js'
import y from './a.js'
setTimeout(()=>{
    console.log(x,y) // 100,20
},100)
```
